# Context Proiect: Site Cabinet Psihologic - Raluca Socaciu

## Descriere
Site de prezentare static pentru cabinetul psihologic al Ralucăi Socaciu din Târgu Mureș.
- **Tehnologie**: HTML/CSS/JS static (fără backend)
- **Hosting**: GitHub Pages (gratuit)
- **URL**: https://marianul.github.io/raluca.socaciu/
- **Repository**: https://github.com/marianul/raluca.socaciu

## Structura Fișierelor

```
raluca.socaciu/
├── index.html                  # Pagina principală (single page)
├── css/
│   ├── style.css               # Stiluri principale + responsive
│   └── blog.css                # Stiluri pentru articole blog

│   ├── servicii.txt            # Lista serviciilor cu iconițe
│   └── despre.md               # Secțiunea Despre Mine (Markdown)
├── images/                     # Imagini (placeholder momentan)
├── blog/
│   ├── article.html            # Template pentru afișare articol
│   └── articole.md             # TOATE articolele într-un singur fișier
└── readme.md                   # Documentație pentru utilizator
```

## Arhitectură Content Loader

Site-ul folosește `js/content-loader.js` pentru a încărca dinamic conținutul din fișierele din `content/`. Acest sistem permite utilizatorului să editeze conținutul fără a cunoaște HTML.

### Funcții principale în content-loader.js:
- `loadContact()` - Încarcă din `content/contact.txt`
- `loadTarife()` - Încarcă din `content/tarife.txt`
- `loadServicii()` - Încarcă din `content/servicii.txt`
- `loadDespre()` - Încarcă din `content/despre.md`
- `loadBlog()` - Încarcă din `blog/articole.md`

### Selectori CSS importanți pentru Contact:
- `.contact-info .contact-item` - 4 elemente în ordine: Adresă, Telefon, Email, Program
- `.footer-contact p:nth-of-type(1/2/3)` - Telefon, Email, Oraș în footer
- `.social-link[aria-label="Facebook/Instagram/LinkedIn"]` - Link-uri social media

### servicii.txt (secțiuni separate cu ---)
```

---
```
- Iconițe: user, users, comments, brain, cloud, video, heart, graduation-cap

### blog/articole.md (toate articolele, separate cu ===)
```markdown
---
title: Titlul Complet al Articolului
date: 2025-01-15
dateFormatted: 15 Ianuarie 2025
category: Categorie
author: Raluca Socaciu
readTime: 5 min
description: Descriere scurtă pentru preview.

---

Conținut articol în Markdown...

```
- **Slug generat automat** din titlu (ex: "Titlul Complet" → "titlul-complet")
- Funcția `generateSlug()` elimină diacritice și caractere speciale
- **Imagini articole**: Câmpul `image` este opțional. Imaginile se pun în `images/blog/`

## Funcționalități Speciale

### EmailJS (Formular Contact)
- Configurare în `js/main.js`:
  - `EMAILJS_PUBLIC_KEY`
  - `EMAILJS_SERVICE_ID`
  - `EMAILJS_TEMPLATE_ID`

### Butoane Programare din Tarife
- Click pe "Programează" din tarife completează automat câmpul mesaj din formular cu serviciul selectat.
- Atribut `data-serviciu` pe butoane.

### Normalizare Line Endings
- Toate funcțiile de parsare folosesc `normalizeLineEndings()` pentru compatibilitate Windows/Unix.

## Considerații Tehnice

### CSS Selectori
- Footer contact: Folosește `nth-of-type` NU `nth-child` (din cauza h4)
- Contact items: Selectate cu `querySelectorAll` și index

### Social Media Links
- Dacă URL-ul e gol sau doar `https://facebook.com/` (fără pagină), link-ul e ascuns
- Logica în `updateSocialLink()`

## Note pentru Dezvoltare

1. **Când modifici structura HTML** în index.html, actualizează selectoarele în content-loader.js
2. **Când adaugi câmpuri noi** în fișierele content, adaugă și logica de încărcare în content-loader.js
3. **Când modifici formatul articolelor**, actualizează și parseArticole() în article.html
4. **Actualizează acest fișier** când faci modificări structurale!
